Справочник "Номенклатура"


Возможности и реквизиты данного справочника отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив поля или элемента интерфейса есть иконка, то элемент доступен только в приложении "1С:Управление нашей фирмой" (1С:УНФ).



Справочник предназначен для хранения информации о товарах, материалах, продукции, полуфабрикатах, объектах вложений в имущество, работах, услугах, технологических операциях и видах работ.

В этой справке:

  • Список номенклатуры
  • Основная информация о номенклатуре
  • Дополнительная информация (характеристики, наборы, партии, серии)

 Важно! Отображение некоторых полей справочника "Номенклатура" также зависит от включенных возможностей программы. Активировать дополнительные возможности можно в разделе Настройки – Еще больше возможностей.



Список номенклатуры

Список номенклатуры реализован как полноценное рабочее место менеджера по продажам.

В верхней части списка номенклатуры можно включить отображение колонок с остатками и ценами.

В правом верхнем углу отображается Корзина, при помощи которой можно легко оформить любой документ, отражающий движение номенклатуры. Для этого достаточно перетащить номенклатуру в корзину и оформить документ. Подробнее о работе с корзиной читайте тут.

Все элементы в справочнике классифицируются по группам и категориям. Список категорий и групп доступен в правой панели. Каждую номенклатурную позицию рекомендуется включать в соответствующую ей Категорию. Это позволит настраивать общие свойства и характеристики категорий и использовать фильтрации номенклатуры по ним.

Загрузите категории номенклатуры из каталога стандартизированных описаний товаров 1С:Номенклатура. В нем собраны основные категории товаров, определены характерные им свойста и самые популярные значения свойств.

Также в списке номенклатуры можно увидеть основное изображение товара. Если изображение не добавлено, его можно добавить прямо из списка товаров. Рядом с изображением отображаются ссылки для работы с аналогами и сопутствующими товарами. При нажатии на ссылку в списке номенклатуры будет установлен отбор по аналогам/сопутствующим товарам.

Быстро находить номенклатуру в списке помогают:

  • поле поиска (Ctrl+F);
  • поиск по штрихкоду с помощью сканера штрихкодов или поля ввода (открывается по клавише F7);
  • фильтры по значениям в верхней панели "Фильтры". Например, можно увидеть только товары, которые есть в наличии, или товары определенного ценового диапазона;
  • фильтры по значениям свойств категорий. Они позволят быстро найти товары, обладающие определенным свойством. Например, отобрать товары определенного производителя.

Настройки фильтров сохраняются для каждого пользователя.

Ввод основной информации о номенклатуре

В карточку номенклатуры заносится следующая информация:

  • Тип. Тип номенклатуры определяет состав реквизитов элемента номенклатуры и порядок использования.
    • Запас. Материальные запасы, подлежащие складскому учету.
    •  Работа. Подрядные работы производственного характера, например, строительство дома, пошив одежды.
    • Услуга. Услуги (сервис), например, туристическое обслуживание, услуги парикмахера, юридические услуги или услуги сторонних организаций, относящиеся на расходы компании, например, аренда помещения или оборудования.
    •  Операция. Технологические операции, выполняемые в процессе выполнения работ, оказания услуг, производстве продукции.
    •  Вид работ. Вид работ, на который может быть задана расценка.
    • Подарочный сертификат.  Физический либо электронный сертификат, который несет в себе некоторый денежный эквивалент.
  • Категория. Укрупненный классификатор номенклатуры, задается отдельным справочником в системе. Каждой категории могут быть назначены общие для всех входящих в нее позиций номенклатуры единицы измерения, свойства и характеристики.
  • Наименование. Используется для отображения в программе.
  • Наименование для печати. Используется для вывода в печатные формы документов. Например: Йогурт "Bio двойная польза" клубника шиповник 480 г - это полное наименование, Bio 2 польза, клубн.шип.480 йогурт - наименование для быстрого поиска.
  • Описание. Произвольное текстовое описание объекта номенклатуры.
  • Единица измерения. Каждая номенклатурная позиция характеризуется учетной единицей измерения. Она выбирается из справочника "Классификатор единиц измерения". Кроме базовой единицы измерения, позиция номенклатуры может иметь необходимое число других единиц измерения (упаковок). Например, сигареты могут иметь единицы измерения: «штука», «пачка», «блок», «коробка». Упаковки могут быть заданы как индивидуально для конкретных позиций номенклатуры в панели навигации элемента, так и для номенклатурной группы в целом. Каждая единица измерения (упаковка) имеет свой коэффициент пересчета относительно базовой единицы.
  • Артикул. Цифровой код товара. Текстовое значение, используется в печатных формах документов.
  • Код. Код элемента номенклатуры.
  • Группа. Ссылка на группу справочника, в которую включен элемент. При ручном изменении реквизита "Группа" происходит перемещение текущего элемента из старой группы в новую.
  • Весовой. Для ситуаций, когда используется обмен с подключаемым оборудованием Offline, предусмотрен реквизит Весовой. Признак устанавливается товарам, которые должны выгружаться и нумероваться особым порядком на весах с печатью этикеток.
  • Запрет дробного количества при подборе. Позволяет запретить ввод дробного количества для товаров, которые продаются только штучно. Настройка не работает при продаже через "старое" РМК (до версии 1.6.25).
  • Габариты. Используются для расчета объема отправлений и планирования доставки.
  • Вес. Используется для расчета суммарного веса отправлений при планировании доставки.

В карточку товара можно добавить произвольный реквизит по ссылке + свой реквизит.

Для номенклатуры типа Запас, Услуга и Работа можно добавить изображения. Для этого нажмите на поле Добавить изображение и выберите файл. Все изображения, прикрепленные к карточке номенклатуры, отображаются на вкладке Файлы.

График Продажи отображает актуальную информацию о продажах товарной позиции за период. График  Цены показывает динамику изменения цен.

Группа Остатки позволяет узнать о наличии запасов на складах компании.

 

Учетная информация

  •  Направление деятельности. При продажах товаров, работ и услуг на указанное направление деятельности будет относиться выручка, себестоимость и НДС с продаж. При отражении расходов, возникающих при поступлении услуг сторонних организаций, указанное направление деятельности будет использоваться по умолчанию для оформления документов.
  • Способ списания. Метод оценки запасов при перемещении, продаже, списании запасов. Доступные методы: "По средней" (взвешенная оценка), "FIFO" (взвешенная оценка). В оперативном учете запасы всегда списываются по средней цене (скользящая оценка, определяемая на момент документа). В регламентной процедуре закрытия месяца происходит корректировка стоимости списания согласно выбранному способу.
  • Срок исполнения (дн.). Стандартный срок выполнения заказа для обещания клиенту. Используется при оформлении заказов покупателей. Используется для номенклатуры типа: Запас, Работа и Услуга (только услуги, выполняемые нашей организацией).
  • Ставка НДС. Ставка НДС по умолчанию, для оформления документов.


Закупка и производство

  • Способ пополнения. Способ пополнения запасов номенклатурной позиции по умолчанию: закупка - приобретение у поставщика, производство - изготовление собственными силами, переработка - производство подрядчиком из материалов компании.
  • Поставщик. Поставщик позиции номенклатуры по умолчанию. Используется для номенклатуры типа: Запас и Услуга (только услуги сторонних организаций).
  • Страна происхождения. Страна, из которой ввезена позиция номенклатуры. Указывается для импортных товаров.
  • ТН ВЭД. Код по классификатору товаров номенклатуры внешнеэкономической деятельности. Используется при оформлении таможенных документов, а также для прослеживаемости партий товаров, ввезенных на таможенную территорию ЕАЭС.
  •  Спецификация. Спецификация изготовления номенклатурной позиции по умолчанию. Перейти к списку спецификаций можно из панели навигации элемента.
  • Срок пополнения (выполнения, дн.). Срок в днях, в течение которого возможно пополнение запаса позиции номенклатуры, выполнение работы или оказание услуги.
  • Норма времени (ч). Норма времени по умолчанию для операции.
  • Гарантийный срок, мес. Срок в месяцах, в течение которого предоставляется гарантия на товар., В течение этого срока покупатель может при выявлении недостатков продукции предъявить претензии поставщику.
  • Выписывать гарантийный талон. При продаже такого товара можно распечатать гарантийный талон.
  • Только поддержка остатка при расчете потребностей. При установке опции, в расчете потребностей для этого запаса не будут анализироваться данные заказов. Учитываются только данные об остатках, указанные в управлении запасами.

 Хранение

  • Склад. Склад хранения по умолчанию.
  •  Ячейка. Ячейка склада хранения по умолчанию.

     

Прайс-лист

 Здесь содержатся свойства, необходимые для формирования актуальных прайс-листов.

  • Ценовая группа. Ценовая группа по умолчанию, к которой относится позиция номенклатуры. Используется для группировки товаров в прайс-листе компании.
  • Порядок. Порядок сортировки строк при формировании прайс-листа. Позиции с одинаковым порядковым номером сортируются по наименованию.
  • Нижняя граница остатков и Верхняя граница остатков. Значения используются для представления в прайс-листах информации об остатках (много, мало). Не используются при расчете потребностей в запасах.
  • Это новинка. При установленном флажке в прайс-листе номенклатура будет отмечена как новинка. Действителен до определяет до какой даты товар остается новинкой.
  • Не использовать в прайс-листах. При включенной опции номенклатура не отображается в прайс-листах.

Дополнительная информация (характеристики, наборы, партии, серии)

Из панели навигации элемента справочника "Номенклатура" можно перейти к дополнительной информации о товаре:

  • Характеристики. Использовать характеристики номенклатуры рекомендуется для учета схожих товаров, отличающихся друг от друга по каким-то определенным параметрам. Заполнение характеристик позволяет хранить товары, например, разного цвета или размера в одной карточке номенклатуры и сократить объем справочника "Номенклатура".
  • Партии. Предназначены для хранения дополнительной аналитики номенклатуры и ведения учета в разрезе этой аналитики. Для номенклатурных позиций, принятых от сторонних контрагентов, учет по партиям обязателен.
  • Серии. Признак, по которому товары отличаются друг от друга: серийный номер, дата производства или дата окончания срока годности. Как правило, эта информация наносится на сам товар или его упаковку при производстве. Серии можно указывать справочно или контролировать по ним остатки
  • Наборы.  Под набором подразумевается несколько различных товаров и услуг, объединенных в группу для удобного оформления совместной продажи товаров. Такой элемент номенклатуры можно использовать только в документах продажи, в других местах выбор его будет запрещен.
  • Комплектации. Варианты комплектов, для которых выполняется фасовка товаров или сборка комплектов номенклатуры.
  •  Спецификации. Варианты состава продукции и технологии ее производства. Спецификации включают в себя материалы и операции, а также информацию об этапах производства.
  • Штрихкоды. Штрихкоды используются для идентификации объектов и быстрого подбора в документы с использованием сканера штрихкодов. Штрихкоды хранятся как строка, состоящая из цифр, букв и символов. Могут быть представлены в графическом виде в печатных формах документов, этикетках и ценниках.
  • Аналоги. Позволяет зарегистрировать  аналоги номенклатурной позиции.
  • Сопутствующие товары. Товары и услуги, косвенно связанные с основным товаров и позволяющие увеличить общий чек продажи.
  • и др.

Закладки в форме номенклатуры появляются только в том случае, если в настройках учета (раздел: Настройки - Еще больше возможностей – Общее) включены соответствующие опции.
