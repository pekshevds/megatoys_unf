Справочник "Контрагенты"


Возможности и реквизиты данного справочника отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив поля или элемента интерфейса есть иконка, то элемент доступен только в приложении "1С:Управление нашей фирмой" (1С:УНФ).



Справочник "Контрагенты" используется для хранения информации о контрагентах. 

Контрагентами являются компании и частные лица: поставщики, покупатели, подрядчики и другие компании, с которыми ваша организация заключает договоры.

В этой справке:

  • Список контрагентов
  • Карточка контрагента
  • Заполнение реквизитов по ИНН или наименованию

 Важно! Отображение некоторых полей справочника "Контрагенты" зависит от включенных возможностей программы. Активировать дополнительные возможности можно в разделе Настройки – Еще больше возможностей.



Список контрагентов

Справочник "Контрагенты" сразу открывается с предустановленными настройками, в зависимости от того, откуда выполнен переход:

  • ссылка Покупатели (раздел Главное, Продажи, а также  CRM , Работы). В списке отображаются все контрагенты, для которых установлен тип взаимоотношений «Покупатель».
  • ссылка Поставщики (раздел Закупки). В списке отображаются все контрагенты, для которых установлен тип взаимоотношений «Поставщик».
  • ссылка Контрагенты (раздел Компания или Настройки – Все справочники). В списке отображаются все контрагенты без установленных отборов.

В правой части списка отображается контактная информация контрагента. Прямо из панели можно позвонить клиенту, написать письмо или посмотреть адрес на карте.

В списке справочника "Контрагенты" можно установить отборы:

  • по основным ролям контрагентов в информационной базе: "Покупатели", "Поставщики", "Прочие". Флажки для установки отбора расположены в верхней части формы списка;
  • по тегам, сегментам, источникам привлечения, контактным лицам и ответственному – отборы расположены в сворачиваемой правой части списка;
  • по группам – отбор осуществляется с помощью выбора созданных пользователем групп контрагентов в блоке Иерархия (группы).

 

Карточка контрагента

Ввод данных о контрагенте осуществляется с помощью формы – карточки контрагента.

В реквизитах группы О контрагенте вводятся название контрагента,  представление в программе и его юридические данные.

Флажками определяется вид взаимоотношений с контрагентом: Покупатель, Поставщик, Прочие. Признак Прочие используется для государственных организаций и банков, например, при отражении перечисления налогов и взносов. Если контрагент одновременно является покупателем и поставщиком можно установить два флажка: Покупатель и Поставщик. В зависимости от вида взаимоотношений контрагенты будут отбираться в списках и формах выбора.

Полное наименование контрагента используется для отражения в печатных формах документов. В поле В программе можно задать краткое название контрагента для выбора и быстрого поиска в информационной базе.

Для каждого контрагента определяется тип организации (юридическое лицо, индивидуальный предприниматель, физическое лицо или государственный орган). Выбор типа влияет на состав реквизитов по ссылке в поле Юридические данные.

Для каждого типа организации предусмотрены поля:

  • для юрлица – страна регистрации, ИНН, КПП, ОКПО, ОГРН для юрлиц, головной контрагент;
  • для ИП – страна регистрации, ФИО, ИНН, ОКПО, ОГРН ИП, номера свидетельства и других сопутствующих данных;
  • для физлица – страна регистрации, ФИО, ИНН, документ, пол и дата рождения;
  • для госоргана – ИНН, КПП, вид и код.

В поле Регистрация устанавливается страна регистрации контрагента. Выбор страны влияет на состав реквизитов в карточке контрагента. Не заполняется для гос. органов.
Если страна, в которой зарегистрирован контрагент, является участником ЕАЭС (в карточке страны установлен флаг "Участник ЕАЭС"), то при выборе контрагента в документах, например "Расходной накладной", будет доступен ввод кода ТНВЭД. Код используется для проведения таможенных операций с прослеживаемыми товарами.

Головной контрагент (для юридических лиц) используется при формировании печатных форм, например, УПД или счет-фактура. Для вывода информации об иерархии по головному контрагенту на форме списка, включите отображение поля Головной контрагент в подменю Еще – Изменить форму.

Тип организации, заполняемый системой по умолчанию, можно настроить для каждого пользователя. Выбранный тип будет подставляться при создании пользователем новой карточки.

В случае некорректного заполнения поля ИНН и других кодов пользователю выводится соответствующее сообщение.

При корректном заполнении ИНН и КПП осуществляется проверка на уникальность. При наличии в базе контрагентов с аналогичной комбинацией ИНН и КПП появляется гиперссылка с информацией об их количестве, по которой можно перейти к списку дублей.

Адреса и телефоны вводятся с использованием специального диалога. Для быстрого ввода российских адресов предусмотрены автоподбор и проверка с помощью адресного классификатора. Загрузка адресных сведений классификатора в информационную базу производится автоматически с портала 1С:ИТС.

В группе Контакты хранится информация о контактных лицах контрагента.

Начните набирать фамилию или имя контакта, в выпадающем списке вам будут предложены подходящие варианты из справочника "Контакты". Если нужного контакта в списке нет, то после заполнения информации о контакте в карточке контрагента новый контакт сохранится в справочнике "Контакты" и "привяжется" к этому контрагенту.

Один контакт может быть связан с несколькими контрагентами. Если контактных лиц несколько, добавьте их по ссылке + контакт. По ссылке + телефон, адрес для каждого контакта можно указать дополнительную информацию – Skype, социальные сети, мессенджеры.

В поле В группе можно определить, в какую группу будет входить текущий контрагент.

Для контрагента можно задать произвольные # Теги – метки, используемые для быстрой классификации контрагентов. Теги задаются в свободной форме. Использование тегов удобно тем, что одному контрагенту можно назначить несколько признаков. По назначенным тегам можно выполнять отбор в справочнике Контрагенты и в отчетах.

В группе Детали указывается общая информация о контрагенте: источник привлечения покупателя, любая дополнительная информация, а также определяется сотрудник, ответственный за работу с контрагентом.

В поле Доступ задается Группа доступа контактов и контрагентов, определяющая ограничения доступа пользователей к элементам справочника и документам по контрагенту.

Статья движения денежных средств используется для автоматического заполнения в банковских и кассовых документах по этому контрагенту.

По ссылке Взаиморасчеты можно выбрать аналитики ведения взаиморасчетов с контрагентом. От выбора зависит количество аналитики в документах, которые нужно заполнить, чтобы получить отчеты по взаиморасчетам с контрагентами в соответствующих разрезах.

Например, долги можно анализировать по договорам, по документам (накладным) или по заказам. Установка флажков приводит к тому, что в документах требуется выбирать договор, документ или заказ.
Установка флажков в колонке "Оплата" позволяет видеть в списке заказов, счетов на оплату или накладных информацию об оплате в виде кружков.

По ссылке + Свой реквизит можно добавить произвольный реквизит для справочника "Контрагенты".

Из карточки контрагента можно вывести на экран список всех документов, связанных с данным контрагентом. Для этого на панели навигации предусмотрена гиперссылка Документы. 

Также из панели навигации можно:

  • перейти к спискам договоров, счетов, контактов, дисконтных карт,  файлов, и  событий данного контрагента (по одноименным гиперссылкам);
  • сформировать акт сверки, отчеты по продажам и взаиморасчетам (гиперссылка Отчеты).

 

Заполнение реквизитов по ИНН или наименованию

Укажите ИНН или наименование контрагента  и нажмите кнопку "Заполнить по ИНН или наименованию"

По кнопке Заполнить по ИНН происходит заполнение реквизитов контрагента на основе данных ЕГРЮЛ и ЕГРИП.

Реквизиты заполняются при наличии информации о юридическом лице или предпринимателе в государственных регистрах.

Для юридического лица:

  • КПП;

  • краткое и полное наименование;

  • руководитель (будет создано контактное лицо);

  • юридический адрес;

  • фактический адрес (если не заполнен);

  • телефон;

Для физического лица (предпринимателя):

  •  краткое, полное наименование;
