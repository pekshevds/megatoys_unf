Справочник "Лиды"

Справочник предназначен для хранения лидов, с которыми ведется работа в компании. 

Лид (англ. lead) – это человек или организация, проявивший интерес к нашему продукту. Для бизнеса это потенциальный клиент, контакты которого необходимо сохранить для последующей работы менеджера.

В этой справке:

  • Список лидов
  • Регистрация лидов
  • Обработка лидов

 

   

 

Список лидов

Форма списка справочника "Лиды" предусматривает два сценария работы : список и доска канбан.

Работа с лидами по методу "канбан" заключается в перетаскивании контакта по доске – таким образом изменяются этапы обработки контакта.

Каждому лиду при этом присваивается состояние, например: не обработан, в работе, выслано коммерческое предложение и т. д. Список состояний формируется пользователем.

Для варианта отображения "список" в правой части списка доступны фильтры по состоянию, тегу, источнику привлечения, ответственному и результату. Также в правой панели отображается контактная информация выбранного лида.

 

Регистрация лидов

Лиды могут быть введены вручную, получены из событий, загружены списком из внешних файлов формата *.xls, *.mxl, *.csv.

В карточке лида указывается информация, характеризующая его:

  • Состояние. Информация о том, на каком этапе сейчас ведется работа с лидом.
  • Наименование. ФИО или название компании.
  • В программе. Представление для выбора и быстрого поиска лида в списках.
  • группа Адреса, телефоны. В группе хранятся адреса и контактная информация для связи с потенциальным клиентом. Для каждого лида можно указать дополнительную информацию – Skype, социальные сети, мессенджеры.
  • В группе. Ссылка на группу справочника, в которую включен элемент. При ручном изменении реквизита "Группа" происходит перемещение текущего элемента из старой группы в новую.
  • # Тег. Для лида можно задать произвольные метки, используемые для быстрой классификации. Теги задаются в свободной форме. По назначенным тегам можно выполнять отбор лидов, например, при подготовке рассылок.
  • группа Детали. Дополнительная информация: источник привлечения лида, а также сотрудник, создавший его.
  • Доступ. Группа доступа лидов, определяющая ограничение доступа пользователей к элементам справочника.

 

Обработка лидов

Когда предварительная работа с лидом завершена, необходимо зафиксировать результат. Обработка лида выполняется в группе Результат с помощью соответствующих кнопок.

Лида можно конвертировать в покупателя, а если намечается сделка, то сразу сформировать заказ.

По кнопке  в правом верхнем углу блока Результат настраивается соответствие полей, данные из которых нужно переносить при конвертации лида в покупателя.

Если лид оказался "некачественным", например, человек просто ошибся номером или хотел совсем не то, что вы предлагаете, это также необходимо отразить.

Нажмите кнопку Некачественный лид и укажите причину завершения работы и, если возможно, комментарий, поясняющий причину отказа.
