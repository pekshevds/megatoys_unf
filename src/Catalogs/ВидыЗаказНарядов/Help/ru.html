Справочник "Виды заказ-нарядов"

Справочник предназначен для группировки различных состояний заказ-нарядов по бизнес-процессам.

Каждый вид заказа будет представлять собой отдельную воронку продаж.

Для ввода дополнительных видов требуется включение функциональной опции "Виды заказ-нарядов".


Описание процессов

Изменения заказов можно можно настроить индивидуально под процессы бизнеса. Например определить, с какого состояния обязательно заполнять причину отмены заказа или сохранять историю изменений.

Сейчас в процессах можно увидеть не только настройки настраиваемые на форме, но и навыки Даши, которые выполняются на определенных состояниях заказа. Таким образом, карта процессов позволит понять, что происходит с заказом в каждом состоянии. 

Состояние выполнения

Заказ может считаться фактически выполненным при переводе его в состояние выполнения. В этом состоянии документ отражает отгрузку всех товаров со склада, завершение работ, начисление зарплаты сотрудникам. При этом на дату выполнения будут совершены фактические движения: списание со склада, реализация товаров, начисление зарплаты.

Корректировка заказа

Корректировка заказа позволяет отметить строки заказа отмененными, к примеру, когда клиент отказался от товара в момент отгрузки или поставщик не привез часть товара. 

Отмененные товары не будут влиять на итоги и конечную сумму документа. Для отмены товара перейдите в документ и в таблице товаров установите флаг "Отменен".

Работу с корректировкой заказов можно донастроить под процессы вашей компании:

1) Процесс "Обязательно заполнять поле "Причина отмены товара". 

Отметьте состояние, начиная с которого заполнение поля "Причина отмены товара" станет обязательным для заполнения.

При проведении заказа, если поле не заполнено, будет выдано предупреждение и проведение заказа будет отменено. 

Заполнение поля "Причина отмены товара" позволит вести детальную аналитику по отмене строк в заказе.

2) Процесc "Запрещать завершение неотгруженных заказов". 

Отметьте вариант завершения заказа ("Завершен: успешно", "Завершен: отменен"), в котором необходимо проверить отгрузку заказа.

Если заказ отгружен не полностью будет выдано предупреждение и проведение заказа будет отменено. 

Вы можете воспользоваться командой "Отменить неотгруженные товары" в подменю "Заполнить" на форме заказа, чтобы удобно отменить все неотгруженные товара заказа. 

После отмены неотгруженных товаров можно корректно завершить работу по заказу.

Общие настройки (для всех видов заказа)

1) Процесс "Запрещать изменение данных прошлых периодов". 

Выберите состояние, в котором при внесении изменений в заказ будет выдано предупреждение и запись/проведение заказа будет отменено. 

Запрет внесения изменений будет распространятся на все виды заказов покупателей.

Более детальную настройку запрета изменения данных вы можете провести перейдя в Настройки - Настройки пользователей и прав - Даты запрета изменений


2) Процесс "Хранить историю изменений (заказ покупателя, заказ-наряд)".

 Включение процесса позволяет хранить все изменения по следующим документам: заказ покупателя, заказ-наряд.

Детальную информацию по изменениям можно будет посмотреть в документе по команде "История изменений".

Хранение истории изменений включается одновременно для всех состояний и видов заказа покупателя, заказ-наряда.

Более детальную настройку запрета изменения данных вы можете провести перейдя в Настройки - Общие настройки - Хранение истории изменений.
