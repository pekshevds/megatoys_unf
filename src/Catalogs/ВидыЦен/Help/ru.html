Справочник "Виды цен"

 

Возможности и реквизиты данного справочника отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив поля или элемента интерфейса есть иконка, то элемент доступен только в приложении "1С:Управление нашей фирмой" (1С:УНФ).


Справочник предназначен для хранения видов отпускных цен, а также учетных цен компании.

В этой справке:

  • Формирование видов цен
  • Прайс-листы
  • Заполнение и расчет цен
    • Расчет динамических цен (процент)
    • Расчет динамических цен по формуле
    • Расчет цен "Произвольный запрос"
  • Загрузка цен из Excel
  • Округление цен
  • Обновление цен

 

Формирование видов цен

Отпускные цены используются для подстановки по умолчанию в документах отгрузки, в заказах покупателей.

Справочник содержит два предопределенных вида цен:

  • Розничная. Используется для подстановки при оформлении розничных продаж с помощью рабочего места кассира (РМК) и документа "Чек ККМ".

  • Учетная цена. Может использоваться, например, как плановая себестоимость номенклатуры.

 Пользователь имеет возможность редактировать предопределенные виды цен и добавлять в справочник любые другие виды цен.

Для каждого вида цены в шапке задаются наименование, тип и правило расчета НДС. Правило расчета НДС определяет, должна ли цена включать в себя НДС.

В поле "Тип" задается один из следующих способов расчета цены:

  • Статический,
  • Динамический (процент),
  • Динамический (формула),
  • Произвольный запрос,
  • Расширение.

Выбор типа сопровождается изменением состава реквизитов формы.

Статические цены (в том числе произвольный запрос и расширение) и динамические цены без авторасчета могут использоваться в рознице, а значит применяться для подразделений с типом "Склад", "Розничный склад", "Розничный склад (суммовой учет)", "Магазин" или группа складов.

 Цены разных видов цен можно сравнивать между собой при помощи отчетов.


 Валюта хранится в карточке вида цен и является общей для всех цен этого вида. В рамках одного вида цен невозможно указать цены в разных валютах. Для динамических цен, которые рассчитываются процентом валюта наследуется из базового типа цен.

Важно! Для динамических видов цен (формула) программа отслеживает изменение курса только для тех валют, у которых включен признак "Вводить вручную".

Для остальных динамических видов цен в валюте пересчет по новому курсу инициализируется пользователем из карточки вида цен по команде "Регистрировать новые цены и закрыть".


 

Прайс-листы

Прайс-листы компании формируются в разрезе видов цен. Виды цен отображаются как колонки прайс-листа. Каждая позиция номенклатуры может иметь столько цен, сколько в системе зарегистрировано видов цен.

По ссылке "Прайс-листы" на панели навигации можно перейти к списку всех прайс-листов, в которых задействован данный вид цен.

 Перейти к справке "Прайс-листы"

 

Заполнение и расчет цен

Цена может указываться вручную, загружаться из внешнего источника (например, файл Excel) или рассчитываться автоматически.

Цены, которые зависят от других называются "Динамическими". Динамические цены рассчитываются по базовым ценам (на процент) или по формуле.

Формирование новых записей для динамических видов цен происходит автоматически если включен флаг автоматического расчета в карточке вида цен.

При назначении динамических цен со снятым флажком "Рассчитывать автоматически" при изменении базовых цен создается и заполняется документ "Установка цен".

Для автоматического заполнения видами цен документа "Установка цен" при вводе на основании используется флаг "Заполнять из документа-основания". Цены будут заполнены по соответствующим им видам цен. Флаг может быть указан для видов цен "Статические", "Произвольный запрос", "Расширение".

 

Расчет динамических цен (процент)

При указании типа цен "Динамический (процент)" в форме появляется обязательный для заполнения реквизит "Базовый вид цен", на основании которого будут рассчитываться новые цены.

Для расчета цены устанавливается процент наценки.

Чтобы вид цены актуализировался автоматически на основе базовых цен и их изменений, необходимо установить флажок "Рассчитывать автоматически при изменении базовых цен".

 

Расчет динамических цен по формуле

Цены могут быть рассчитаны по заранее заданной формуле.

При указании способа расчета цен "Динамический (формула)" в форме появляется строка для ввода формулы. По нажатию на пиктограмму рядом со строкой вызывается форма "Редактирование формулы".

Чтобы вид цены актуализировался автоматически на основе базовых цен и их изменений, необходимо установить флажок "Рассчитывать автоматически при изменении базовых цен".

Для цен с типом "Динамический (Процент)" и "Динамический (формула)" можно сразу из формы выполнить регистрацию.

 

Расчет цен "Произвольный запрос"

Цены могут быть рассчитаны по заранее подготовленной схеме компоновки данных, этот вид цен называется "Произвольный запрос".

Конструктор схемы может быть запущен только в режиме "Толстого клиента".

Для удобства работы добавлены предопределенные виды произвольных запросов, такие как "Минимальные цены поставщика", "Максимальные цены поставщика", "Себестоимость" и другие. Данный вид цен может быть назначен только в документе "Установка цен" и не является динамическим.

Для произвольной схемы запроса должны быть указаны следующие реквизиты:

  • Период - тип Дата;
  •  Валюта - тип СправочникСсылка.Валюты;
  • Номенклатура - тип СправочникСсылка.Номенклатура;
  • Характеристика - тип СправочникСсылка.ХарактеристикиНоменклатуры;
  • Цена - тип Число;
  • ЕдиницаИзмерения - составной тип СправочникСсылка.КлассификаторЕдиницИзмерения, СправочникСсылка.ЕдиницыИзмерения.

Цены, сформированные схемой компоновки данных, не являются динамическими.

Загрузка цен из Excel

Вы можете загрузить цены из внешнего источника (например, файла Excel).

 Перейти к справке "Загрузка данных из внешнего источника"

 

Особенности загрузки цен из внешнего источника

Для корректной загрузки цен колонка "Вид цен" в файле должна быть сопоставлена с одним из видов цен в программе.

Если колонки "Вид цен" нет в файле и не удалось найти вид цен по наименованию, будет подставлено значение "по умолчанию". 

При загрузке цен из формы списка "Виды цен", в качестве загружаемого вида цен "по умолчанию" принимается текущая строка, которая была активна в момент нажатия кнопки загрузки в списке.

Если значение "по умолчанию" не задано, будет подставлен вид цен из настроек пользователя. 

Вид цен "по умолчанию" отображается на последнем шаге мастера загрузки. При необходимости, пользователь может его изменить.

 

Округление цен

Для цены может быть заполнен формат, правило округления и психологическое округление.

Психологическое округление  при формировании цены вычитает определенную сумму, чтобы  создать у клиента впечатления сниженной цены.

 Можно установить сумму для вычета из имеющейся цены и диапазон цен для проведения округления. Настроенное психологическое округление позволит автоматически поддерживать привлекательные цены для покупателей, тем самым увеличивая продажи.

"Психологическое округление" применяется после основного округления, например, 100 рублей -> 99 рубль 99 копеек (при значении вычитаемой суммы = 0.01)

Результат округлений можно проверить на форме в поле "Пример".

 Перейти к статье "Психологическое округление цены"

 

Формат цен

По ссылке "Формат цены" можно настроить формат для выбранного вида цен. Настройка выполняется в специальном диалоге – "Конструкторе форматной строки", который вызывается по нажатию гиперссылки реквизита "Формат цены".

Формат используется при выводе цены в прайс-листе.

 

Обновление цен

Цены с типом "Динамический (Процент)" и "Динамический (формула)" могут актуализироваться автоматически на основе базовых цен и их изменений, если установлен флажок "Рассчитывать автоматически при изменении базовых цен".

При актуализации динамических цен со снятым флажком "Рассчитывать автоматически" при изменении базовых цен создается и заполняется документ "Установка цен".

Для цены могут быть заданы пороги пересчета как общие так и по ценовым группам.

Например, если предыдущая цена была 1000, а новая будет 1200, то процент изменения составит +16,66%.

Если при  этом указать общий порог увеличения (или по ценовой группе) 20%, то при заполнении документа "Установка цен" на основании "Приходной накладной" или расчете динамических цен - цена в регистр сведений "Цены" записана не будет, так как не превышен указанный порог.
