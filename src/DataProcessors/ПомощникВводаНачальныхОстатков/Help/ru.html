Обработка "Помощник ввода начальных остатков"


Возможности и реквизиты данного инструмента отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив возможности или элемента интерфейса есть иконка, то описанная опция доступна только в приложении "1С:Управление нашей фирмой" (1С:УНФ).


 

Заполнение остатков производится при помощи обработки "Помощник ввода начальных остатков", позволяющего ввести остатки для основных разделов учета и сформировать документы "Ввод начальных остатков". Данный документ предназначен для хранения всех входящих (начальных) остатков.

Помощник предлагает пользователю выполнить пять шагов, за которые вводятся данные по всем разделам учета:

  • Деньги,
  • Товары,
  • Расчеты,
  •  Прочее,
  • Финиш.

Перед началом заполнения необходимо определить дату ввода начальных остатков. Для остатков по всем разделам учета дата будет единой.

Дату ввода остатков можно изменить в сформированном документе, при этом все остатки будут перемещены на новую дату.

Для остатков товаров, расчетов с поставщиками и расчетов с покупателями возможна загрузка из "1С:Розницы 2.3", "1С:Управления торговлей 10.3", "1С:Бухгалтерии 3.0", электронных таблиц (например, Microsoft Office Excel, OpenOffice Calc и др.).

Данные о товарах, ценах и остатки также могут быть загружены с сайта, созданного на CMS 1C-Битрикс или 1C-UMI.


Шаг 1 – Деньги

Пользователь вносит в систему информацию об остатках денежных средств на счетах, в кассах организации и у подотчетных лиц.

Из формы можно перейти к созданию нового счета, кассы, подотчетного лица.


Шаг 2 – Товары

Пользователь вносит информацию по остаткам товаров.

Для товаров сразу устанавливаются "Себестоимость" и "Цена продажи". По кнопке "Рассчитать" цены продажи по формуле доступен конструктор формул, позволяющий настроить формулу расчета цены продажи.

Для заполнения таблицы товаров можно использовать сканер штрихкодов, ТСД, загрузку из электронных таблиц.


Шаг 3 – Расчеты

Пользователь отражает состояние взаиморасчетов с поставщиками, покупателями, персоналом, а также долги/авансы по налогам и эквайрингу.

Все расчеты также можно загрузить из внешних источников.


  Шаг 4 – Прочее

На этом шаге пользователь может внести остатки по прочим разделам учета, которые не попали в предыдущие разделы. Например,  имущество и  начисленную амортизацию.
 


Шаг 5 (4) – Финиш

На последнем шаге выполняется формирование документов ввода начальных остатков.

В случае если остатки по какому-либо разделу уже были введены ранее, система предложит продолжить ввод остатков. Введенные данные отобразятся в форме помощника и будут доступны для редактирования. Дата ввода остатков останется прежней.

 

 

  Как проверить правильность ввода остатков?

После ввода начальных остатков по всем разделам учета и организациям, можно выполнить проверку правильности их ввода. Для этого подходят отчеты:

  •  Оборотно-сальдовая ведомость. Отчет позволяет проверить конечные остатки (сальдо) по вспомогательному счету "Служебный". На дату начала учета этот счет не должен иметь каких либо остатков. Оборотно-сальдовая ведомость формируется из раздела Компания - Отчеты.

  •  Баланс. Актив должен быть равен пассиву. В случае отклонения необходимо проверить корректность и полноту заполнения документов ввода начальных остатков. Баланс формируется из раздела Компания - Отчеты.

После первоначальной настройки, ввода начальных остатков и проверка корректности их ввода можно приступать к непосредственному ведению учета в конфигурации.
