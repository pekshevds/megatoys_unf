Обработка "Загрузка данных из УТ 10.3"

С помощью этой обработки можно загрузить данные из конфигурации Управление торговлей, редакция 10.3.

Файлы, необходимые для выгрузки данных из конфигурации "Управление торговлей, ред. 10.3", входят в поставку программы "Управление нашей фирмой 3.0"/"Розница 3.0" и находятся в каталоге ConvTr103.

  • Перенос данных УТ 10.3 - Розница 3.0, УНФ 3.0.epf – внешняя обработка выгрузки данных во внешний файл из конфигурации "Управление торговлей, редакции 10.3".

 

Важно! Перенос данных из конфигурации "Управление торговлей", редакция 10.3, возможен, начиная с версии 10.3.12.

 

Порядок переноса данных:

  1. Подготовка информационной базы 1С:УТ 10.3 к переносу.
  2. Выгрузка данных из информационной базы УТ 10.3.
  3. Загрузка данных в информационную базу 1С:УНФ 3.0 / 1С:Розница 3.0.

 

 

1. Подготовка информационной базы 1С:УТ 10.3 к переносу

Выполните резервное копирование информационной базы конфигурации "Управление торговлей" редакции 10.3. Если версия конфигурации "Управление торговлей" ниже 10.3.12, то проведите обновление конфигурации до указанной или более поздней версии.

Выполните все регламентные процедуры в конфигурации "Управление торговлей", редакция 10.3.

  • Проведите инвентаризацию товарных остатков и остатков денежных средств. Устраните отрицательные данные по остаткам товаров и остаткам денежных средств.
  • Устраните отрицательные остатки взаиморасчетов с подотчетными лицами.
  • Удалите помеченные на удаление объекты.
  • Закройте кассовые смены и инкассируйте полностью денежные средства из контрольно-кассовых машин.
  • Оформите документы "Отчет комиссионера о продажах" и "Отчет комитенту о продажах".
  • Оформите межфирменные продажи между организациями.
  • Выполните проведение документов по партиям.

На момент переноса данных в информационной базе "Управление торговлей", редакция 10.3 не должно быть отрицательных остатков товаров по складам и организациям. Информация об отрицательных остатках товаров не будет выгружена из информационной базы конфигурации.

 

2. Выгрузка данных из информационной базы УТ 10.3

Откройте внешнюю обработку "Перенос данных УТ 10.3 - Розница 3.0, УНФ 3.0.epf" с помощью пункта меню Файл"-"Открыть".

В появившемся диалоге укажите имя внешнего файла, в который будет производиться выгрузка данных. Укажите дату, на которую актуальны остатки товаров, денежных средств и взаиморасчетов.

В диалоговом окне появится список объектов, которые могут быть выгружены. По умолчанию будет выгружена информация по настройкам параметров учета, справочникам и остаткам, рекомендуемым к выгрузке. При необходимости можно отключить выгрузку информации по любому из справочников или отключить выгрузку остатков по любому из регистров конфигурации.

Выгрузка остатков задолженности клиентов производится с учетом конкретных документов реализации или авансов, перечисленных по заказам клиентов.

Документы, по которым есть непогашенная задолженность, не выгружаются в первоначальном виде. В 1С:УНФ на их основании генерируются документы "Ввод остатков" с соответствующим видом операции, с указанием суммы непогашенной задолженности.

После настройки правил выгрузки данных необходимо нажать на кнопку "Выгрузить". При выгрузке данных осуществляется проверка на наличие отрицательных остатков товаров и выдается соответствующее сообщение о тех товарах, по которым обнаружены отрицательные остатки. При необходимости эти данные необходимо откорректировать. После того, как все объекты исправлены необходимо повторно нажать на кнопку "Выгрузить". Данные будут выгружены во внешний файл.

 

3. Загрузка данных в 1С:УНФ 3.0 / 1С:Розница 3.0

Создайте новую информационную базу конфигурации "1С:Управление нашей фирмой", редакция 3.0 / "1С:Розница", редакция 3.0 локально или в облаке. Данные из файла должны загружаться в новую, не заполненную
информационную базу.

При первом запуске откроется стартовое окно. В данном окне необходимо заполнить данные об организации.

Данные организации в 1С:УНФ 3.0 должны быть заполнены также как конфигурации "Управление торговлей 10.3" с точностью до символа. Используйте копирование для переноса данных полей.

  • Поле "Наименование" перенесите в "Название";
  • поле "ИНН" перенесите в "ИНН".

Это необходимо для сопоставления предопределенной организации в 1С:УНФ с организацией, которую переносим из 1С:УТ 10.3.

Перейдите в раздел Компания – Начальные остатки – Загрузка данных из УТ 10.3. В открывшемся диалоговом окне укажите внешний файл, в который была произведена выгрузка, и нажмите на кнопку "Загрузить". Данные будут загружены в информационную базу из внешнего файла.

После проведения загрузки убедитесь, что данные загружены корректно. Остатки товаров, денежных средств и взаиморасчетов фиксируются документом "Ввод остатков". С помощью отчетов проверьте правильность выгрузки остатков в обеих базах.

При повторной загрузке остатков данные в документах ввода остатков, созданных конвертацией, перезаполняются заново.
