Документ "Отчет комиссионера"


Возможности и реквизиты данного документа отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив возможности или элемента интерфейса есть иконка, то описанная опция доступна только в приложении "1С:Управление нашей фирмой" (1С:УНФ).



Документ предназначен для хранения информации о продажах запасов, которые были переданы нашей организацией на реализацию комиссионеру.

С помощью данного документа комиссионер (контрагент) отчитывается перед комитентом (нашей организацией) о реализованных товарах. По данным отчета рассчитывается сумма комиссионного вознаграждения за проделанную работу.

В этой справке:

  • Заполнение документа
  • Оплата
  • Расчет комиссионного вознаграждения
  • Отчет комиссионера о возвратах

 

 Важно! Отображение некоторых полей справочника "Номенклатура" зависит от включенных возможностей программы. Активировать дополнительные возможности можно в разделе Настройки – Еще больше возможностей.


 

Заполнение документа

Документ "Отчет комиссионера" предоставляется комиссионером  по итогам реализации товаров, переданных ему на комиссию, или по истечению определенного периода (например, ежемесячно).

Условия сотрудничества с комиссионером отражаются в карточке договора (вид "С комиссионером") в группе "Комиссия" и используются для заполнения отчета комиссионера.

  Перейти к справке "Договоры"

 

С помощью документа могут быть зафиксированы действия с комиссионными товарами по операциям:

  • Продажа. Соответствует хозяйственной операции "Отчет комиссионера о продажах". Фиксирует  факт продажи комиссионером запасов. 
  • Возврат. Соответствует хозяйственной операции "Отчет комиссионера о возвратах". Фиксирует  факт возврата товаров покупателем комиссионеру .

Реквизиты документа:

  • Организация. Собственная организация, от имени которой оформляется документ. Выбирается из справочника "Организации". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.
  • Контрагент. В данном реквизите указывается комиссионер. Выбирается из справочника "Контрагенты".
  • Договор. Указывается договор контрагента с видом "С комиссионером", в рамках которого оформляется документ. Выбирается из справочника "Договоры контрагентов".

По гиперссылке "Редактировать цены и валюту" вызывается диалог редактирования видов цен данного документа.

На закладке "Реализация" вводится информация о покупателях и товарах, проданных комиссионером.

В  табличной части "Покупатели" комитент регистрирует информацию о покупателях и датах реализации. При записи документа по строкам, в которых установлен флаг "Выписан счет-фактура" автоматически создаются счета-фактуры.

Реквизиты таблицы "Покупатели":

  • Покупатель - покупатель запасов комитента. Выбирается из справочника "Контрагенты".
  • Всего - стоимость запасов, проданных комитентом покупателю. Реквизит рассчитывается автоматически на основании данных табличной части "Запасы".
  • Выставлен СФ - признак необходимости автоматической регистрации счет-фактуры. (для операции "Продажа")
  • Дата СФ - дата реализации покупателю запасов комитента. (для операции "Продажа")
  • Счет-фактура - ссылка на выставленный счет-фактуру. (для операции "Продажа")

Табличная часть "Реализация" подчиняется табличной части "Покупатели". Одной строке табличной части "Покупатели" может соответствовать несколько строк табличной части "Запасы".

Табличная часть "Реализация" может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части) либо методом ручного ввода.

Реквизиты таблицы "Реализация":

  • Номенклатура - позиция номенклатуры, за реализацию которой комиссионер отчитывается перед нашей организацией. Выбирается из справочника "Номенклатура" с типом "Запас".
  • Характеристика - при необходимости в данном поле указывается характеристика номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.
  • Партия - при необходимости в данном поле указывается партия номенклатуры. Выбирается из справочника "Партии номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение партионного учета.
  • Количество - количество реализованной номенклатуры, за которую отчитывается комиссионер.
  • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".
  • Цена - цена, по которой номенклатурная позиция была реализована комиссионером.
  • Сумма - стоимость номенклатуры по данной строке.
  • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".
  • Сумма НДС - сумма налога на добавленную стоимость по данной строке.
  • Всего - поле, автоматически заполняемое суммарной стоимостью запасов, которые были реализованы комиссионером, согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".
  • Цена передачи - цена, по которой номенклатурная позиция была передана комиссионеру.
  • Сумма передачи - стоимость, по которой номенклатурная позиция была передана комиссионеру. 
  • Сумма НДС передачи - сумма налога на добавленную стоимость с суммы передачи по данной строке.
  • Сумма вознаграждения - сумма комиссионного вознаграждения, рассчитанная исходя из условий договора с комиссионером и параметров закладки "Дополнительно".
  • Сумма НДС вознаграждения - сумма налога на добавленную стоимость с суммы вознаграждения по данной строке.
  • Указать заказ - признак того будет ли заказ покупателя указан вручную или подобран автоматически при распределении.
  • Заказ - заказ покупателя, в рамках которого передавались запасы комиссионеру. Выбирается документ "Заказ покупателя". Реквизит необязателен для заполнения.
  • Страна происхождения - страна происхождения запасов.
  • Номер ГТД или РНПТ- грузовая таможенная декларация запаса или регистрационный номер партии товара.

На закладке "Возвраты" вводится информация о покупателях и товарах, проданных комиссионером, но по которым покупатель осуществил возврат. 

В  табличной части "Покупатели возвраты" комитент регистрирует информацию о покупателях, датах реализации и по какому Отчету комиссионера происходит возврат. При записи документа по строкам, в которых установлен флаг "Выписан счет-фактура" автоматически создаются счета-фактуры.

Реквизиты таблицы "Покупатели возвраты": 

  • Покупатель - покупатель запасов комитента. Выбирается из справочника "Контрагенты".
  • Всего - стоимость запасов, проданных комитентом покупателю. Реквизит рассчитывается автоматически на основании данных табличной части "Запасы".
  • Выставлен СФ - признак необходимости автоматической регистрации счет-фактуры. (для операции "Продажа")
  • Дата СФ - дата реализации покупателю запасов комитента. (для операции "Продажа")
  • Счет-фактура - ссылка на выставленный счет-фактуру. (для операции "Продажа")
  • Отчет комиссионера - ссылка на Отчет комиссионера по которому осуществляется возврат.

Табличная часть "Возвраты" подчиняется табличной части "Покупатели возвраты". Одной строке табличной части "Покупатели возвраты" может соответствовать несколько строк табличной части "Запасы". 

Табличная часть "Возвраты" может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части) либо методом ручного ввода.

На закладке  "Дополнительно" пользователь может внести информацию:

  • Подразделение, ответственное за операцию. Выбирается из справочника "Структурные единицы" с типом "Административно-управленческое подразделение". Реквизит необязателен для заполнения.

  • Проект, в рамках которого товары передавались на комиссию. Выбирается из справочника "Проекты". Реквизит необязателен для заполнения.
  • Ответственный за ввод текущего документа. Выбирается из справочника "Сотрудники". В реквизит автоматически подставляется автор документа.
  • Комментарий. Поле ввода, предназначенное для указания любой дополнительной информации к документу.

 

Оплата

На закладке "Предоплата" заполняются сведения по зачету авансов, полученных ранее от комиссионера.

Реквизиты табличной части:

  • Документ - документ поступления аванса от покупателя: "Поступление в кассу", "Поступление на счет".
  • Заказ - здесь можно сослаться на документ "Заказ покупателя".

  • Сумма расчетов - сумма в валюте договора.
  •  Курс - курс пересчета в валюту документа.

  •  Кратность - кратность пересчета в валюту документа.
  •  Сумма платежа - указывается в валюте документа.

По кнопке "Подобрать" вызывается диалог "Зачет предоплаты"  для подбора авансов покупателей в табличную часть документа.

 

Расчет комиссионного вознаграждения

Расчет комиссионного вознаграждения выполняется на закладке "Дополнительно" если установлен флажок "Удержать комиссионное вознаграждение".

Реквизиты для расчета вознаграждения:

  • Способ расчета комиссионного вознаграждения. Способ расчета комиссионного вознаграждения выбирается из выпадающего списка. Доступны варианты:

    • Не рассчитывается (такой способ расчета означает, что сумма комиссионного вознаграждения указывается индивидуально).
    • Процент от суммы продажи (под суммой продаж понимается та сумма, по которой комиссионер продал запасы).
    • Процент от разности сумм продажи и поступления (под суммой продаж понимается та сумма, по которой комиссионер продал запасы, а под суммой поступления понимается та сумма, по которой запасы были переданы ему на реализацию).
  • Процент комиссионного вознаграждения. Указывается процент для расчета комиссионного вознаграждения.
  • Ставка НДС с комиссионного вознаграждения. Указывается ставка налога на добавленную стоимость с комиссионного вознаграждения.
