Документ "Комплектация запасов"


Возможности и реквизиты данного документа отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив поля или элемента интерфейса есть иконка, то элемент доступен только в приложении "1С:Управление нашей фирмой" (1С:УНФ).



Документ предназначен для отражения единичной операции комплектации товаров на складе: фасовки товаров и сборки комплектов номенклатуры.

С помощью документа также могут быть зафиксирована операция разборки товара на комплектующие.

 

В этой справке:

  • Особенности учета комплектов
  • Заполнение документа "Комплектация"

 

 Важно! Отображение некоторых полей документа зависит от включенных возможностей программы. Активировать дополнительные возможности можно в разделе Настройки – Еще больше возможностей.


 

Особенности учета комплектов

Комплекты, в отличие от наборов, собираются и учитываются на складе до продажи как самостоятельный товар.

Остатки комплектов на складах можно узнать не только из специальных отчетов – они будут отображаться в справочнике "Номенклатура", форме подбора и документах продажи в режиме "Показать остатки и резервы".

Номенклатура комплекта не требует установки каких-либо признаков. По ссылке "Комплектация" на командной панели элемента справочника "Номенклатура" вводятся варианты комплектации для товара.

 Перейти к справке справочника "Комплектации"

 

Заполнение документа "Комплектация"

В документе доступны две операции:

  • Сборка – комплектация товаров на складе. В шапке указывается собираемый запас. В табличной части "Товары" указываются израсходованные комплектующие.
  • Разборка – разборка исходного запаса. В шапке указывается исходная номенклатура, которую необходимо разобрать. В табличной части "Товары" указывается выходной продукт, т. е. результат разборки.

В документе может отражаться сборка только одной номенклатурной позиции в одном варианте комплектации, количество комплектов при этом не ограничено. То есть для каждого нового варианта комплектации необходимо вводить отдельный документ.

В документе необходимо заполнить следующие реквизиты:

  • Номенклатура. В данное поле вводится элемент справочника номенклатуры.
  • Характеристика. При необходимости в данном поле указывается характеристика номенклатурной позиции. Поле доступно только в том случае, если в настройках учета указано ведение учета по характеристикам номенклатуры. Выбирается из справочника "Характеристики номенклатуры".
  • Партия. Указывается партия номенклатурной позиции. Поле доступно только в том случае, если в настройках учета указано ведение учета по партиям номенклатуры. Выбирается из справочника "Партии". 
  • Ед. Единица измерения, в которой вводится количество номенклатуры. Поле доступно только в том случае, если в настройках включен учет в нескольких единицах измерения.
  • Количество. Указывается количество единиц измерения номенклатурной позиции. 
  • Страна происхождения номенклатурной позиции. Выбирается из справочника "Страны мира".
  • Номер ГТД. Указывается номер грузовой таможенной декларации. Выбирается из справочника "Номера ГТД".
  • Организация. В данном реквизите указывается собственная организация, от имени которой оформляется документ.
  • Операция. Вид операции документа.
  • Склад. В этом поле необходимо указать структурную единицу, из которой будут списаны запасы. В отличие от документов производства, комплектация может выполняться на розничных складах без предварительного перемещения.
  •  Ячейка. В этом поле необходимо указать ячейку, из которой будут списаны запасы.

Варианты комплектации пользователь выбирает из одноименного справочника, предназначенного для хранения информации о составе комплектов.

Состав комплекта заполняется на закладке "Товары". Заполнение состава выполняется по выбранному в шапке  варианту комплектации с помощью кнопки "Заполнить".

Также для заполнения состава комплекта в документе реализованы подбор, загрузка из файлов, копирование строк.

Реквизиты табличной части "Товары":

  • Номенклатура. В данное поле вводится элемент справочника номенклатуры.
  • Характеристика. При необходимости в данном поле указывается характеристика номенклатурной позиции.
  • Партия. Указывается партия номенклатурной позиции.
  • Ед. Единица измерения, в которой вводится количество номенклатуры. 
  • Количество. Указывается количество единиц измерения номенклатурной позиции. 
  • Доля стоимости. Используется при разборке для расчета пропорции разделения себестоимости исходного комплекта между комплектующими.
  • Страна происхождения номенклатурной позиции. Выбирается из справочника "Страны мира".

  • Номер ГТД. Указывается номер грузовой таможенной декларации. Выбирается из справочника "Номера ГТД".

После сборки комплект можно продавать как товар, а также использовать для сборки других комплектов.

При использовании ордерного склада на основании документа "Комплектация запасов" вводится документ "Приходный ордер" или "Расходный ордер" (в зависимости от операции).
