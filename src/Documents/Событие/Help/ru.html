Документ "Событие"

Документ "Событие" предназначен для регистрации и планирования в программе взаимодействий с клиентами. Также используется для отправки электронной почты и загрузки входящих писем.

В этой справке:

  • Типы событий
  • Работа с cобытиями


         


Типы событий

События могут быть следующих типов:

  • личная встреча,
  • телефонный звонок,
  • электронное письмо,
  • SMS,
  • прочее.

При создании нового события от выбранного типа зависит состав заполняемых реквизитов.

Для типов событий "Личная встреча", "Телефонный звонок", "Прочее" заполняются следующие реквизиты:

  • Состояние. В данном реквизите указывается категория классификации события. Выбирается из справочника "Состояния событий". Реквизит обязателен для заполнения.

  • Контрагент. В данном реквизите указывается информация о контрагенте. Выбирается из справочника "Контрагенты". Реквизит необязателен для заполнения.
  • Тема. В данном реквизите указывается основное описание события. Реквизит необязателен для заполнения.
  • Содержание. В данном реквизите указывается описание события. Реквизит необязателен для заполнения.
  • Документ основание. В данном реквизите указывается основание события. Реквизит необязателен для заполнения.
  • Начало события. В данном реквизите указывается дата и время начало события. Реквизит необязателен для заполнения.
  • Окончание события. В данном реквизите указывается дата и время окончания события. Реквизит необязателен для заполнения.
  • Ответственный. В данном реквизите указывается информация о сотруднике, ответственном за ввод текущего документа. Выбирается из справочника "Сотрудники". Реквизит необязателен для заполнения.
  • Участники. Данная табличная часть содержит перечень участников события и контактную информацию участника.
     

Событие "Телефонный звонок"

В программе можно подключить IP-телефонию для работы с входящими и исходящими телефонными звонками. Телефонные звонки фиксируются документом "Событие" с типом "Телефонный звонок".

При поступлении входящего звонка в программе может быть автоматически создано событие.

Совершить звонок клиенту можно из события, карточки контрагента, или из любого списка, в котором присутствует правая панель с контактными данными контрагента.

Звонок будет сохранен в программе, даже если разговор не состоялся. Если оператор телефонии поддерживает запись телефонных разговоров, ссылка на запись также будет доступна в событии-звонке.

 Перейти к главе руководства "Телефонные звонки"

 

Событие "Электронное письмо"

Получение, хранение и отправка электронных писем в программе реализована с помощью событий типа "Электронное письмо".

Для электронного письма заполняются e-mail одного или нескольких получателей, тема и содержание письма, добавляются, если необходимо, вложения.

Далее указывается учетная запись электронной почты, с которой будет осуществляться отправка.

В случае успешной отправки письма автоматически устанавливается состояние "Завершено".

 Перейти к справке "Электронное письмо"

 

Событие "SMS"

Отправка SMS в программе реализована с помощью событий типа "SMS". Внешний вид события данного типа похож на событие "Электронное письмо".

В шапке указывается список адресатов-получателей, задается текст сообщения. Тема сообщения используется только для быстрого заполнения содержания или тематического разделения рассылок и не входит в отправляемое сообщение.

Рядом с колонкой "Номера телефонов" есть поле "Статус доставки". Оно заполняется автоматически и сигнализирует о том, в каком состоянии находится SMS-сообщение, отправленное соответствующему адресату.
По кнопке "Обновить статусы доставки" статусы обновляются сведениями, полученными от оператора связи.

 Перейти к справке "SMS"

 

Работа с событиями

Все события, связанные с контрагентом, доступны непосредственно из его карточки (справочник "Контрагенты"). Пользователь в любой момент может ознакомиться с историей взаимодействия с конкретным контрагентом по гиперссылке "События". 

В форме события сохраняется информация о событии: тема, содержание, список контактов участников. Каждому событию присваивается одно из трех возможных состояний: Запланировано, Завершено, Отменено. 

Для события устанавливается важность, указываются даты его начала и окончания – планируемого или фактического. Если заполнить для события поле "Календарь", оно будет отображаться в календаре сотрудника. Календари выбираются из одноименного справочника. 

Можно также указать значения дополнительных реквизитов по гиперссылке "+ Свой реквизит". 

События в программе создаются пользователями вручную или записываются автоматически в результате работы с инструментами взаимодействия: электронной почтой, телефонией, массовыми рассылками. 

Пользователь может продолжить работу с клиентом из формы события. Например, набрать номер контакта, запланировать новое событие, создать контрагента, контакт, лид или документ ("Заказ покупателя", "Заказ-наряд", "Заказ поставщику" и другие).
