Документ "Заказ поставщику"

 

Возможности и реквизиты данного документа отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив поля или элемента интерфейса есть иконка, то элемент доступен только в приложении "1С:Управление нашей фирмой" (1С:УНФ).



Документ предназначен для оформления намерений организации приобрести товары и услуги у поставщика.

Заказ может являться документом, на основании которого производятся оплата и получение товара.

 

В этой справке:

  • Список "Заказы поставщикам"
  • Заполнение заказа поставщику
  • Планирование и зачет оплаты
  • Ввод на основании

 

 Важно! Отображение некоторых полей документа "Заказ поставщику" зависит от включенных возможностей программы. Активировать дополнительные возможности можно в разделе Настройки – Еще больше возможностей.

 

 

Список "Заказы поставщикам"

Список содержит две вкладки. На вкладке "Заказы поставщикам" отображается информация о выполнении заказов, состоянии заказов, о проценте оплаты по ним.

Информация о выполнении отражается оформлением строк заказов:

  • серая строка означает, что заказ завершен;

  • зачеркнутая – что заказ отменен.

Степень отгрузки и оплаты отображается с помощью круговых пиктограмм. Пиктограмма в первом столбце списка заказов отражает состояние отгрузки по заказу, во втором столбце – состояние оплаты по заказу. Справа в меню настройки можно установить отбор по состоянию отгрузки или оплаты.

На вкладке "Заказы покупателей" отображаются незакрытые заказы покупателей, товары для которых необходимо заказать у поставщиков. В список попадают только заказы покупателей, по которым еще не сформированы заказы поставщику, содержащие запасы и услуги, для которых установлен способ пополнения "Закупка" или  "Переработка".

По кнопке "Оформить заказ поставщику" на основании заказа покупателя формируется заказ поставщику.

Для создания единого заказ поставщику по нескольким заказам покупателя необходимо выделить в списке одновременно несколько заказов. Это можно сделать с зажатой клавишей CTRL или Shift.

 

Заполнение заказа поставщику

При оформлении документа "Заказ поставщику" указываются состояние, дата исполнения заказа, контрагент - поставщик товаров и договор, в рамках которого оформляется документ.

В поле "Организация" указывается собственная организация, от имени которой оформляется документ. Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.

В зависимости от установленного вида операции с помощью данного документа может быть оформлен заказ на поставку или заказ на переработку.

На форме документа расположена гиперссылка "Цены и валюта", по которой вызывается диалог редактирования видов цен, скидок данного документа, а также  валюты.

В табличное поле вводятся данные о заказываемой номенклатуре. Таблица может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части) либо методом ручного ввода.

Реквизиты табличной части:

  • Номенклатура - позиция номенклатуры, на которую оформляется заказ.

  • Характеристика - при необходимости в данном поле указывается характеристика номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

  • Количество - количество номенклатурной позиции.
  • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

  • Цена - цена поставки номенклатурной позиции.

  • % Скидки - процент ручной скидки (наценки).

  • Сумма - стоимость заказываемой номенклатуры по данной строке.

  • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

  • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

  • Всего - поле, автоматически заполняемое суммарной стоимостью запасов согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

  • Дата поступления. Ожидаемая дата поступления.

  • Заказ покупателя,  для выполнения которого будут использоваться заказанные у поставщика товары (т. е. поместить заказ покупателя в заказ поставщику). Колонка отображается при переключении расположения заказа покупателя в документах поступления на значение "В табличной части".

  • Содержание - представление товара, услуги в печатной форме. Чтобы реквизит стал видимым, необходимо выполнить "Все действия" - "Изменить форму" и установить флаг напротив реквизита.

Положение реквизита "Заказ покупателя" можно выбрать "в шапке" или "в табличной части". Для выбора нескольких заказов покупателя, под который осуществляется поставка номенклатуры в документе, необходимо переключить отображение поля на значение "В табличной части". Отображение поля настраивается с помощью команды "Шапка/табличная часть" в контекстном меню или по кнопке "Еще;.

В подвале документа можно указать ручную скидку на документ, процентом или суммой.

 

Планирование оплаты по заказам

Для заказов в работе или заказов, имеющих состояние "Завершен", можно запланировать оплату, установив флажок "Запланировать оплату" на закладке "Платежный календарь".

На закладке пользователь указывает планируемые даты, способ оплат и процент, который должен быть выплачен не позднее указанной даты.

Суммы оплаты (процента) и НДС будут рассчитаны автоматически.

Оплату заказа поставщику можно запланировать в несколько этапов. Для этого установите в нажатое состояние кнопку "Список" в правом верхнем углу. Данные об оплате теперь вводятся списком.

 

Ввод на основании

Документ "Заказ поставщику" может быть введен на основании:

  • справочников"Контрагенты" и "Договоры контрагентов";
  • документов "Заказ покупателя",
  • "Счет на оплату (полученный)",
  •  "Заказ-наряд",
  •  "Событие",
  •  "Заказ на производство".
