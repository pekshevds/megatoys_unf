Документ "Акт выполненных работ"


Возможности программы отличаются в зависимости от используемой конфигурации: "1С:УНФ" или "1С:Розница". 
Документ "Акт выполненных работ" доступен только в приложении "1С:Управление нашей фирмой" (1С:УНФ).



Документ предназначен для хранения информации по отражению в учете реализации работ и услуг сторонним контрагентам.

По содержанию документ  аналогичен расходной накладной, однако в табличной части указывается только номенклатура с типом "Работа" или "Услуга".

В этой справке:

  • Список документов "Акт выполненных работ"
  • Заполнение документа
  • Закладка "Работы и услуги"
  • Закладка "Оплата"

 Важно! Отображение некоторых полей документа "Акт выполненных работ" зависит от включенных возможностей программы. Активировать дополнительные возможности можно в разделе Настройки – Еще больше возможностей.


   


Список документов "Акт выполненных работ"

Список документов содержит информацию о выполнении работ и степени оплаты по документу.

Пиктограмма "Состояние оплаты"  не выводится для непроведенных документов.

В списке можно отфильтровать документы по состоянию оплаты:

  • Без оплаты;
  • Оплачен частично;
  • Оплачен полностью;
  • Есть переплата.

Фильтр не учитывает документы контрагентов, для которых не ведется учет оплаты по документам (настраивается в карточке контрагента).

Закладка "Заказы покупателей (к отгрузке)" в списке документов позволяет отражать факт выполнения услуг покупателям по находящимся в работе заказам покупателей. Из данной формы можно сформировать "Акт выполненных работ" по одному или нескольким выделенным заказам покупателей.

Документ также может быть сформирован на основании документа "Заказ на производство".


Заполнение документа

В шапке документов необходимо заполнить следующие реквизиты:

  • Организация. В данном реквизите указывается собственная организация, в которой выполняется операция. Выбирается из справочника "Организации". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.

  • Заказ покупателя. Выбирается документ "Заказ покупателя". Есть возможность сменить расположение реквизита с помощью команды "Шапка / Табличная часть". Реквизит не обязателен для заполнения.

  • Заказ на производство. Выбирается документ "Заказ на производство". Есть возможность сменить расположение реквизита с помощью команды "Шапка / Табличная часть". Реквизит не обязателен для заполнения.

  • Контрагент. В данном реквизите указывается покупатель. Выбирается из справочника "Контрагенты".

  • Договор. В данном реквизите указывается договор контрагента, в рамках которого оформляется документ. Выбирается из справочника "Договоры контрагентов".

  • Подразделение. Выбирается из справочника "Структурные единицы" с типом "Подразделение".

Для реквизитов "Заказ покупателя" и "Заказ на производство" есть возможность сменить расположение реквизита с помощью команды "Шапка / Табличная часть". Вариант "В табличной части" используется в случае, когда акт формируется сразу по нескольким заказам.

На форме документа расположена гиперссылка "Редактировать цены и валюту", по которой вызывается диалог редактирования видов цен, скидок и валюты данного документа.

В подвале формы можно указать ручную скидку на документ, процентом или суммой.

В реквизите Основание указывается "Счет на оплату" или "Прием в ремонт", на основании которого сформирован данный документ. Реквизит не обязателен для заполнения.

 

Закладка "Работы и услуги"

Табличная часть "Работы и услуги" может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части), сканированием штрихкода или методом ручного ввода.

Реквизиты табличной части:

  • Номенклатура - позиция номенклатуры, которая реализуется. Выбирается из справочника "Номенклатура" с типом "Услуга" или "Работа".

  • Характеристика - при необходимости в данном поле указывается характеристика реализуемой номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

  • Количество - количество реализуемой номенклатурной позиции.

  • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

  • Цена - цена реализуемой номенклатурной позиции.

  • % Скидки - процент ручной скидки (наценки).

  • % Авт. - процент автоматической скидки (наценки). Процент автоматической скидки редактируется в справочнике "Виды автоматических скидок и наценок". Расчёт автоматических скидок в документе осуществляется при записи документа и при нажатии кнопки "% Авт." над табличной частью.

  • Сумма - стоимость реализуемой номенклатуры по данной строке.

  • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

  • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

  • Всего - поле, автоматически заполняемое суммарной стоимостью услуг или работ согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

  • Содержание - представление работы (услуги) в печатной форме.

При включенном контроле минимальных цен продажи (и установленных минимальных ценах товара) в табличной части цвет текста поля цена будет изменяться на красный, если значение цены установлено меньше чем минимальная.
Минимальные цены подбираются из общего вида минимальных цен.


Закладка "Оплата"

Закладка "Оплата" заполняется сведениями по зачету оплаты от покупателя или по зачету авансов, полученных ранее.

Порядок зачета оплаты: автоматически или вручную – определяется настройками в заказе, в договоре с покупателем, в карточке клиента и общими настройками системы.

Переключатель "Режим работы" определяет режим работы вкладки.

В режиме "Редактирование" отображается табличная часть, в которой можно вручную добавить документы оплаты. Для быстрого заполнения таблицы оплат предусмотрены кнопки:

  • Помощник. Открывает форму "Помощник зачета предоплаты", предназначенную для ввода сведений о зачете авансов, полученных ранее от покупателя.

  • Подобрать. Открывает форму "Зачет предоплаты", в которой можно выбрать документы к зачету.

Режим "Помощник" покажет остаток долга контрагента, а также какие платежи этого контрагента еще не соотнесены ни с одной накладной. В этом окне пользователь сможет сразу распределить деньги между заказами и накладными контрагента.

В шапке помощника выводится сводная информация по документу, из которого он открыт.

Чтобы зачесть предоплату, необходимо отметить флажками документы оплаты или воспользоваться командой "Заполнить по остаткам". Перенести результаты работы помощника в заказ покупателя можно по кнопке "Перенести в документ".

Вкладка переключится в режим работы "Отчет". В нем отобразится информация о зачтенных оплатах по данному заказу.
