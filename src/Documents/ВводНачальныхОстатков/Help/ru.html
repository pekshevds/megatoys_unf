Документ "Ввод начальных остатков"


Возможности и реквизиты данного документа отличаются в приложениях "1С:УНФ" и "1С:Розница". 
Если в справке напротив возможности или элемента интерфейса есть иконка, то описанная опция доступна только в приложении "1С:Управление нашей фирмой" (1С:УНФ).


Документ предназначен для хранения всех входящих (начальных) остатков.

Заполнение документа "Ввод начальных остатков" осуществляется в разрезе разделов учета. Раздел учета отображается в поле реквизита "Раздел учета". В зависимости от выбранного раздела изменяется состав данных и форма документа.

В документе доступны следующие разделы учета:

  • Запасы,
  • Денежные средства,
  • Расчеты с поставщиками и покупателями,
  • Расчеты с подотчетниками,
  • Расчеты с персоналом,
  • Расчеты по налогам,
  • Эквайринг,
  •  Имущество,
  • Прочие разделы. Используется для ввода данных, для которых не выделен отдельный раздел учета.

 

Заполнение остатков производится при помощи обработки "Помощник ввода начальных остатков". Инструмент позволяет быстро ввести остатки для основных разделов учета и сформировать документы "Ввод начальных остатков".

Помощник предлагает пользователю выполнить пять шагов, за которые вводятся данные по всем разделам учета:

  • Деньги,
  • Товары,
  • Расчеты,
  •  Прочее,
  • Финиш.

Для остатков товаров, расчетов с поставщиками и расчетов с покупателями возможна загрузка из "1С:Розницы 2.3", "1С:Управления торговлей 10.3", "1С:Бухгалтерии 3.0", электронных таблиц (например, Microsoft Office Excel, OpenOffice Calc и др.). Данные о товарах, ценах и остатки также могут быть загружены с сайта, созданного на CMS 1C-Битрикс или 1C-UMI.

На последнем шаге  обработки "Помощник ввода начальных остатков" выполняется формирование документов ввода начальных остатков.

В случае если остатки по какому-либо разделу уже были введены ранее, система предложит продолжить ввод остатков. Введенные данные отобразятся в форме помощника и будут доступны для редактирования. Дата ввода остатков останется прежней.

  Перейти к справке "Помощник ввода начальных остатков"

 

  Как проверить правильность ввода остатков?

После ввода начальных остатков по всем разделам учета и организациям, можно выполнить проверку правильности их ввода. Для этого подходят отчеты:

  •   Оборотно-сальдовая ведомость. Отчет позволяет проверить конечные остатки (сальдо) по вспомогательному счету "Служебный". На дату начала учета этот счет не должен иметь каких либо остатков. Оборотно-сальдовая ведомость формируется из раздела Компания - Отчеты.

  •  Баланс. Актив должен быть равен пассиву. В случае отклонения необходимо проверить корректность и полноту заполнения документов ввода начальных остатков. Баланс формируется из раздела Компания - Отчеты.

После первоначальной настройки, ввода начальных остатков и проверка корректности их ввода можно приступать к непосредственному ведению учета в конфигурации.
