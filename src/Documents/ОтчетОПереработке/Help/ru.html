Документ "Отчет о переработке"


Возможности программы отличаются в зависимости от используемой конфигурации: "1С:УНФ" или "1С:Розница". 
Документ "Отчет о переработке" доступен только в приложении "1С:Управление нашей фирмой" (1С:УНФ).



Документ предназначен для хранения информации об отчетах перед заказчиком за переработку принятого сырья и материалов.

 

Необходимо заполнить следующие реквизиты:

  • Организация. В данном реквизите указывается собственная организация, которая принимала запасы в переработку. Выбирается из справочника "Организации". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.

  • Заказ покупателя. В данном реквизите указывается заказ, в рамках которого принимались запасы в переработку. Выбирается документ "Заказ покупателя". Реквизит необязателен для заполнения.

  • Склад. В данном реквизите указывается структурная единица, с которой отгружается результат переработки. Выбирается из справочника "Структурные единицы" с типом "Склад".

  • Ячейка. В данном реквизите указывается ячейка структурной единицы (стеллаж, участок), с которого отгружается результат переработки. Выбирается из справочника "Ячейки". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по ячеистым складам.

  • Контрагент. В данном реквизите указывается давалец сырья и материалов в переработку. Выбирается из справочника "Контрагенты".

  • Договор. В данном реквизите указывается договор контрагента, в рамках которого оформляется документ. Выбирается из справочника "Договоры контрагентов".

 

На форме документа расположена гиперссылка "Редактировать цены и валюту", по которой вызывается диалог редактирования видов цен, скидок и валюты данного документа.

В подвале формы можно указать ручную скидку на документ, процентом или суммой.

  • Продукция. Данная табличная часть может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части) либо методом ручного ввода. Реквизиты табличной части:

    • Номенклатура - позиция номенклатуры, представляющая собой результат переработке давальческого сырья и материалов. Выбирается из справочника "Номенклатура" с типом "Запас".

    • Характеристика - при необходимости в данном поле указывается характеристика номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

    • Партия - при необходимости в данном поле указывается партия номенклатуры. Выбирается из справочника "Партии номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение партионного учета.

    • Количество - количество изготовленной номенклатуры.

    • Резерв - количество зарезервированной номенклатурной позиции.

    • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

    • Цена - цена изготовленной номенклатуры.

    • % Скидки - процент ручной скидки (наценки).

    • % Авт. - процент автоматической скидки (наценки). Процент автоматической скидки редактируется в справочнике "Виды автоматических скидок и наценок". Расчёт автоматических скидок в документе осуществляется при записи документа и при нажатии кнопки "% Авт." над табличной частью.

      Сумма - стоимость номенклатуры по данной строке.

    • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

    • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

    • Всего - поле, автоматически заполняемое суммарной стоимостью запасов, которые были изготовлены организацией, согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

    • Спецификация - спецификация, содержащая сведения о технологии производимой продукции. Выбирается из справочника "Спецификации". Реквизит необязателен для заполнения.

    • Содержание - представление номенклатуры в печатной форме. Чтобы реквизит стал видимым, необходимо выполнить "Все действия" - "Изменить форму" и установить флаг напротив реквизита.

  • Материалы. Данная табличная часть может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части) либо методом ручного ввода. Реквизиты табличной части:

    • Номенклатура - позиция номенклатуры, представляющая собой отнесенные материальные затраты (материалы и сырье давальца) на изготовление продукции и полуфабрикатов. Выбирается из справочника "Номенклатура" с типом "Запас".

    • Характеристика - при необходимости в данном поле указывается характеристика номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

    • Партия - партия номенклатуры, по которой номенклатурная позиция принималась в переработку. Выбирается из справочника "Партии номенклатуры".

    • Количество - количество отнесенной номенклатуры на изготовление продукции и полуфабрикатов.

    • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

    • Цена - цена материальных затрат (материалы и сырье давальца).

    • Сумма - стоимость номенклатуры по данной строке.

    • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

    • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

    • Всего - поле, автоматически заполняемое суммарной стоимостью материальных затрат (материалы и сырье давальца), согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

  • Отходы. Данная табличная часть может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части) либо методом ручного ввода. Реквизиты табличной части:

    • Номенклатура - позиция номенклатуры, представляющая собой отход, возникший в результате изготовления продукции и полуфабрикатов. Выбирается из справочника "Номенклатура" с типом "Запас".

    • Характеристика - при необходимости в данном поле указывается характеристика номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

    • Партия - при необходимости в данном поле указывается партия номенклатуры. Выбирается из справочника "Партии номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение партионного учета.

    • Количество - количество отходов.

    • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

  • Предоплата. Данная табличная часть заполняется сведениями по зачету авансов, полученных ранее от давальца. На форме документа расположена гиперссылка "Редактировать зачет предоплаты", по которой вызывается диалог редактирования табличной части "Зачет предоплаты". Реквизиты табличной части:

    • Документ - документ поступления аванса от покупателя: "Поступление в кассу", "Поступление на счет".

    • Сумма расчетов - сумма в валюте договора.

    • Курс - курс пересчета в валюту документа.

    • Кратность - кратность пересчета в валюту документа.

    • Сумма платежа - указывается в валюте документа.

  • Дополнительная информация. Группа реквизитов:

    • Подразделение - информация о подразделении, ответственном за операцию. Выбирается из справочника "Структурные единицы" с типом "Административно-управленческое подразделение". Реквизит необязателен для заполнения.

    • Ответственный - информация о сотруднике, ответственном за ввод текущего документа. Выбирается из справочника "Сотрудники". Реквизит необязателен для заполнения.

    • Реквизиты печати - дополнительные сведения, которые используются при заполнении печатных форм.


Минимальные цены

При включении опции "Минимальные цены продажи" и настройки минимальных цен (Раздел "Продажи" -> "Еще больше возможностей") в табличной части цвет текста поля цена будет изменен на красный, если значение цены станет меньше чем минимальная. Минимальные цены продажи будут подбираться по складу или общие если склад не указан.
